A smooth first day depends on a complete account, not a half provisioned one. A checklist turns onboarding from a scramble into a routine.

## Create the identity

- Create the user in Active Directory in the correct organizational unit
- Set a temporary password and require a change at first sign in
- Match the username and display name to the naming standard

## Grant the right access

- Add the user to the security groups for their role, not one by one but by the role template
- Assign the correct Microsoft 365 license and confirm the mailbox is created
- Add shared mailboxes, distribution lists, and drive mappings the role needs

## Verify before handoff

Sign in as the user, or have them sign in while you watch, and confirm mail, drives, and key apps all work. Catching a missing group now beats a frantic ticket at nine on Monday.

## Record it

Keep the checklist with the ticket. The same template drives offboarding in reverse.
